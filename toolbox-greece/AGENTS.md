# AGENTS.md

- 日本語で簡潔に応答する。
- 推測より確認を優先する。
- 作業前に Git の状態を確認し、必要に応じて `codex/` で始まる作業ブランチへ分けてから変更する。
- 変更は `cwd` 配下に限定し、必要最小限にとどめる。
- `cwd` 外や破壊的操作は事前確認する。
- 秘密情報や実行時データは追加しない。
- 長い作業や複数エージェント作業では、途中再開できるように目的、現在地、完了済み、未完了、次の一手を短く残す。
- 複雑な作業では、オーケストレーターがチェックリストで進捗を管理し、完了・進行中・未着手を更新する。
- 役割分担を明示したいときは Zeus または `$zeus-orchestrate` を入口にする。
- 基本は着手前に `moira`（`apps/moira` の CLI）で `.ai/moira.json` に目的・現在地・次の一手・決定ログ・タスク状態を残してから進める。`moira` が使えない環境やごく短い作業では、インラインのチェックリストで代替してよい。
- 中断後の再開時は、最初に `moira show` を読み、完了済みは Git と突き合わせてから次の一手に進む。
- `.ai/` は実行時データなのでコミットしない（各プロジェクトのローカル状態）。
