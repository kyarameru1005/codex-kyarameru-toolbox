# agents

このディレクトリは、研究用の開発ハーネスで使うサブエージェント定義を置く場所です。
各 `*.toml` は 1 つのエージェント設定を表し、名前、役割、使用モデル、推論強度、sandbox 権限、開発者向け指示を持ちます。

## 役割一覧

| ファイル | 主な役割 | 名前の由来 |
| --- | --- | --- |
| `zeus.toml` | 全体判断を行うマネージャ | ゼウスは神々の王で、全体を統括して判断する役割に合わせています。 |
| `hermes.toml` | 実装前の調査 | ヘルメスは伝令神で、情報を運び、関連箇所を素早く見つける調査役に合わせています。 |
| `daedalus.toml` | 設計・アーキテクチャ整理 | ダイダロスは名工で、構造や仕組みを設計する役割に合わせています。 |
| `hephaestus.toml` | 実装担当 | ヘパイストスは鍛冶神で、設計を実際の成果物へ作り上げる役割に合わせています。 |
| `athena.toml` | 実装後レビュー | アテナは知恵と戦略の女神で、実装を冷静に見直すレビュー役に合わせています。 |
| `themis.toml` | テスト・評価 | テミスは法と秩序の女神で、結果を基準に照らして評価する役割に合わせています。 |
| `ares.toml` | セキュリティ・リスク確認 | アレスは戦の神で、危険や攻撃面を意識してリスクを見る役割に合わせています。 |
| `apollo.toml` | README や設計メモなどの文書化 | アポロンは知性や芸術を司る神で、内容を分かりやすく整える文書化役に合わせています。 |
| `chronos.toml` | ログ整理と振り返り | クロノスは時間を連想させる名で、時系列ログや履歴を整理する役割に合わせています。 |

## ファイルの見方

各設定ファイルには主に次の項目があります。

- `name`: エージェント名
- `description`: 短い役割説明
- `model`: 使用モデル
- `model_reasoning_effort`: 推論の強さ
- `sandbox_mode`: `read-only` / `workspace-write` などの実行権限
- `developer_instructions`: そのエージェント専用の行動指針と出力形式

## モデル選定

トークン効率と安定性のバランスを取るため、役割ごとにモデルを分けます。

- `gpt-5.4`: 判断ミスの影響が大きい、全体判断、設計、実装、レビュー、検証、リスク確認に使います。
- `gpt-5.4-mini`: 調査、文書化、ログ整理のように、範囲を絞って短く処理しやすい作業に使います。
- `model_reasoning_effort = "high"`: 設計、レビュー、セキュリティなど、見落としのコストが高い役割に限定します。
- `model_reasoning_effort = "medium"`: 通常の判断、実装、調査、文書化、ログ整理に使います。

## 運用の考え方

- `Zeus` は最初に使う判断役です。軽い整理は自分で行い、必要に応じて他エージェントの利用を決めます。
- `Hermes` → `Daedalus` → `Hephaestus` の順で、調査・設計・実装を分離できます。
- 実装後は `Athena` や `Themis` で、レビューと検証を分けて扱えます。
- 文書化や研究記録が必要な場合は `Apollo` と `Chronos` を使います。

## 注意点

- `workspace-write` のエージェントは書き込み可能なので、実装や文書更新を担います。
- `read-only` のエージェントは調査、判断、レビュー向けです。
- 役割分離が前提なので、定義を変えるときは責務が混ざらないか確認してください。
