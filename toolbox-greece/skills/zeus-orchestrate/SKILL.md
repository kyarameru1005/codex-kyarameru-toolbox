# zeus-orchestrate

Zeus を入口にして、作業を始める前に役割分担と進め方を決めるために使う。

## 起動方法

ユーザーが次のように明示したら、このスキルを使う。

```text
$zeus-orchestrate を使って進めてください。
```

目的、前提、完了条件が書かれている場合は、それを優先する。
不足していても、合理的に読み取れる範囲で進める。

## 役割

- まず Zeus が必要な支援を判断する。
- 関連ファイルや影響範囲が不明なら Hermes または `labyrinth-explore` を使う。
- 設計判断が必要なら Daedalus または `oracle-design` を使う。
- 実装が必要なら Hephaestus または `forge-implement` を使う。
- 実装後は Athena、Themis、`aegis-review`、`gauntlet-verify` のうち必要な確認を入れる。
- セキュリティ、危険操作、秘密情報の懸念があれば Ares を使う。
- README、設計メモ、作業記録が必要なら Apollo、Chronos、`chronicle-docs` を使う。
- 他エージェントへ仕事を振った場合は、最後に誰へ何を依頼したかと、その結果をどう使ったかを報告する。

## 手順

1. 目的、前提、完了条件を短く整理する。
2. ファイル変更を伴う作業では、最初に Git 状態を確認する。
3. 必要な役割だけを選び、使う理由と使わない理由を短く示す。
4. 複雑な作業では、調査、設計、実装、レビュー、検証、報告から必要な項目だけでチェックリストを作る。
5. 実装へ進む場合は、関連ファイルと変更方針が明確になってから進める。
6. 完了前に検証結果と未確認事項を整理する。
7. 他エージェントを使った場合は、最終報告に担当者、依頼内容、結果の利用方法を含める。

## 出力

- 判断
- 理由
- 進行計画
- 必要な場合だけタスクチェックリスト
- 人間が実装前に理解すべきこと
- 他エージェントを使った場合だけ委譲記録
