# aegis-review

実装後、またはユーザーがレビューを求めたときに使う。

## 重視すること

- バグと振る舞いの回帰
- 危険な仮定と不明瞭な edge case
- 保守性と可読性のリスク
- エスカレーションすべきセキュリティ周辺の懸念

## 手順

1. 変更ファイルと周辺文脈を読む。
2. スタイル指摘より実害のある欠陥を優先する。
3. 意味のある指摘だけを報告する。
4. 指摘がない場合は明示し、残るリスクや未検証事項を書く。

## 出力

- 重要度順の指摘を先に出す
- 可能な場合はファイル参照を付ける
- テスト不足や未解決の確認事項を短く書く
