# gauntlet-verify

実装後、またはユーザーが検証内容を求めたときに使う。

## 重視すること

- 最小で価値の高い確認
- 成功、失敗、未検証の分離
- 過剰に断定しない失敗原因の解釈
- 完了判断前に不足している証拠を見つけること

## 手順

1. 変更内容に合う確認項目を特定する。
2. 最も関連性の高い証拠を実行または確認する。
3. 結果を pass、fail、unverified として整理する。
4. カバレッジが不十分なら、次に有用な確認を示す。

## 出力

- 検証対象
- 使用したコマンドまたは証拠
- 結果の要約
- 残っている不足や未確認事項
