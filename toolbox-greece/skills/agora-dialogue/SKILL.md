# agora-dialogue

別の AI と、共有ファイルを介して非同期に会話するための作業パターン。
Codex が依頼やレビュー要求を書き、別 AI（例: Claude）が同じファイルに応答を書き、Codex がそれを読んで取り込む。

このスキルは **ファイルの受け渡しに閉じる**。別 AI への実際の送信（CLI / MCP の実行）は範囲外で、人手や別の仕組みで行う。

## 重視すること

- 発言の境界を明確にする（どの発言が誰のものか必ず分かる）
- 1ファイル1トピックに絞る
- 依頼は再現可能な粒度（背景・対象・具体的な問い・受け入れ条件）で書く
- 相手の発言は改変しない
- 秘密情報・実行時データを会話ファイルに書かない

## ファイルの場所と命名

- `.ai/dialogue/<topic>.md` に置く（`.ai/` は gitignore 済みの実行時データ。コミットしない）。
- `<topic>` は短い lowercase hyphen（例: `review-auth-refactor`）。
- 話題が変わるなら新しいファイルにする。

## ファイル形式

Markdown。先頭にメタ、以降は発言ブロックを時系列で追記する。

```
# dialogue: <topic>

- 目的: <なぜこの会話をするか / 何が欲しいか>
- 参加者: codex, claude
- 状態: open / waiting-claude / waiting-codex / done
- 更新: <YYYY-MM-DD>

## [codex → claude] <要求の見出し>
### 背景
### レビュー / 相談の対象
（対象ファイルや差分の範囲。長い場合はパスと該当箇所で指す）
### 具体的な問い
-
### 受け入れ条件 / 観点
-

## [claude → codex] <応答の見出し>
（別 AI が追記する応答。Codex は読み取るだけで編集しない）

## [codex] 取り込み結果
（応答をどう反映したか、未対応・保留を書く）
```

## 手順

作成:
1. `.ai/dialogue/<topic>.md` を作り、メタ（目的・参加者・状態・更新）を書く。
2. `[codex → claude]` ブロックに依頼を書き、状態を `waiting-claude` にする。

読み取り:
3. 相手が `[claude → codex]` を追記したら、その部分を読む。相手の発言は書き換えない。

編集（追記）:
4. Codex は自分の発言ブロックだけを追記する。相手の発言は改変・要約・削除しない（引用は自分のブロック内で行う）。
5. 反映したら `[codex] 取り込み結果` を書き、状態を `done` か次の `waiting-*` に更新する。

## 注意

- 相手（他 AI）の発言ブロックは編集しない。読み取り専用として扱う。
- 秘密情報・認証情報・大きな生成物は会話ファイルに貼らない（パスや要約で示す）。
- 会話ファイルはコミットしない（`.ai/` 配下、実行時データ）。

## 出力

- 作成 / 更新したファイルのパスと現在の状態
- 相手に渡す依頼の要点、または読み取った応答の要点
- 取り込み結果と未対応事項
