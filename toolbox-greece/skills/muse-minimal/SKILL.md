# muse-minimal

引き算で本質だけを残す系統。
`muse-interface`（共通の UI 構造設計）の上に、ミニマルの見た目トーンを与える。

## この系統の狙い / 与える印象

- 明快・集中・無駄のなさ。1画面の主役が1つ分かる
- 装飾ではなく構造と余白で整える

## 配色・トーン

- ほぼモノクローム（白・黒・グレー）。色は機能（リンク/警告）にのみ最小限
- 面の塗りより線と文字で構成

## タイポグラフィ

- 字種は1つ。サイズ段階も少なく（2〜3段）
- 太さの差で階層を作る

## レイアウト・余白

- 要素数を絞る。余白が最大の装飾
- 罫線や区切りも最小限にし、間隔で区切る

## 装飾・質感・モーション

- 影・グラデ・角丸はほぼ使わない（使うなら統一して極小）
- モーションは即時か、ごく短いフェードのみ

## 避けること（アンチパターン）

- 装飾の足し算、複数アクセント色
- 情報の詰め込み、過剰な区切り線

## 出力（Hephaestus に渡す見た目指針）

- モノクローム配色と機能色の最小範囲
- 字種1つ・サイズ段階の少なさ・太さでの階層付け
- 余白主体で区切る方針（線を増やさない）
