# muse-interface

すぐ実装することより、見た目と操作体験の設計判断の質が重要なときに使う。
`oracle-design` がソフトウェア構造の設計なのに対し、こちらは UI/ビジュアルの設計を担う。

UI 設計の親スキル。共通の UI 構造（レイアウト・情報設計・状態表現・一貫性・アクセシビリティ）を決める。
見た目のトーン（高級感・モダンなど）は、スタイル系統スキル（`muse-modern` / `muse-luxury` / `muse-minimal` / `muse-bold` / `muse-avantgarde`）を併用して与える。

## 重視すること

- 情報設計と視覚階層（何を一番に見せるか）
- コンポーネント構成と再利用
- 状態表現（空・読み込み中・エラー・成功）
- 既存パターンとの一貫性
- アクセシビリティ（コントラスト、フォーカス、代替テキスト、操作可能性）

## 手順

1. 対象画面と利用者の目的、主要な操作を明確にする。
2. 既存の UI パターン・命名・トーンを確認し、それに合わせる。
3. 現実的な少数の案を視覚階層と状態を含めて比較する。
4. 要件を満たす最も単純なデザインを優先する。
5. 実装（Hephaestus）に渡せる粒度で要素・配置・状態を書く。

## 出力

- 推奨方針
- 採用しなかった選択肢と短い理由
- 主要要素・配置・状態を含む構成
- アクセシビリティで満たすべき点
- 次の実装に向けた指針
