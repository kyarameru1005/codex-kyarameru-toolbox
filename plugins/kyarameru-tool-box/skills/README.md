# skills

このディレクトリは、作業手順を再利用するためのスキル定義を置く場所です。
このプラグインでは `agents/` と同じ北欧神話名を使い、どの役割の入口かを名前だけで分かるようにしています。

## 役割との対応

- `odin`: 作業開始時に進め方と役割分担を決める
- `heimdall`: 関連ファイル、依存関係、変更波及を調べる
- `mimir`: 構造、責務、選択肢を整理して設計を固める
- `thor`: 方針が固まった変更を最小差分で実装する
- `forseti`: 実装差分をレビューして危険点を洗う
- `tyr`: テストや確認結果を整理して評価する

## 使い分け

- 入口が必要なときは `odin`
- 調査が先なら `heimdall`
- 実装前に判断材料を揃えるなら `mimir`
- 変更対象が明確なら `thor`
- 実装後の防御的確認には `forseti`
- 完了前の確認整理には `tyr`

## 書き方のルール

- 1 スキル 1 目的に絞る
- frontmatter の `name` はディレクトリ名と一致させる
- `description` は「何の入口か」が即座に分かる文にする
- 本文は目的、観点、手順、出力のいずれかを短く定義する
- エージェント固有の人格説明ではなく、作業の進め方に寄せる

## 追加・変更時の注意

- `agents/` に同名の役割があるか確認する
- README、`AGENTS.md`、テストの参照名を同時に更新する
- 名前だけ変えて責務がずれる状態を作らない
