# agents

このディレクトリは、北欧神話ベースの役割名で分けたサブエージェント定義を置く場所です。
各 `.toml` は、担当責務、推論強度、sandbox 権限をまとめて定義します。

## 使い方

最初に `odin` が全体を見て、必要なら専門エージェントへ切り分けます。
1 つのエージェントに複数責務を持たせず、調査、設計、実装、レビュー、検証を明確に分ける前提です。

## 既定の役割

- `odin.toml`: 全体判断と役割分担を行う管理役
- `heimdall.toml`: 関連ファイル、処理フロー、影響範囲を調査する役
- `mimir.toml`: 構造、責務分離、選択肢比較を整理する設計役
- `thor.toml`: 方針が固まった変更を最小差分で実装する役
- `forseti.toml`: 実装差分をレビューし、バグや危険な前提を洗う役
- `tyr.toml`: テスト方針と確認結果を整理して検証する役

## 選び方

- 何から着手するか曖昧なときは `odin`
- 編集対象や影響範囲が見えていないときは `heimdall`
- 実装前に構造を決めたいときは `mimir`
- 方針が固まっていて手を動かす段階なら `thor`
- 差分の危険点を洗いたいときは `forseti`
- 実行した確認を PASS / FAIL / 未確認 で整理したいときは `tyr`

## 設計ルール

- 命名は北欧神話ベースで統一する
- `description` は役割を 1 文で言い切る
- `developer_instructions` には責務だけを書き、手順は増やしすぎない
- `sandbox_mode` は最小権限にする
- 実装しない役は `read-only` を維持する

## 追加・変更時の注意

- 新しいエージェントを足す前に、既存 6 役で分離できない理由を明確にする
- `skills/` 側にも対応する入口が必要か確認する
- README、`AGENTS.md`、テストの期待値を同時に更新して命名のずれを防ぐ
