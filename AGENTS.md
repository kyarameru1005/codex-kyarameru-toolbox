# Project AGENTS Policy

このファイルは `kyarameru-tool-box` リポジトリ専用ルールを定義する。

## 目的 / 優先順位

- 目的は、`~/.codex` へ配備するツールボックスを安全に維持し、実装・検証・レビューの判断基準を固定することとする。
- 優先順位は `正確性 > 安全性 > 速度` とする。

## 応答ルール

- 応答は原則日本語とし、英語で依頼された場合のみ英語で返す。
- 回答は `結論 -> 根拠 -> 次アクション` の順で簡潔に示す。
- 不確かな情報は断定せず、現状コード、ドキュメント、実行結果を根拠として示す。

## 実行ルール

- 実装前に `AGENTS.md`、`README.md`、`docs/` の順で関連情報を確認する。
- 推測で変更せず、現状コードと既存スクリプトを根拠に判断する。
- 変更後は関連する検証コマンドを実行し、成功結果または未実行理由を報告する。
- 外部反映を伴う操作を行う場合は、実施内容を明示してから実行する。

## リポジトリ構造

- リポジトリ構造の詳細は `docs/repository-layout.md` を参照する。
- 本書では、変更判断に必要な運用方針のみ扱う。

## 実行コマンド

- 変更後は、リポジトリで定義済みのテスト・検証・配備確認コマンドを実行する。
- コマンド一覧の詳細は `README.md` と `docs/` 配下の正本文書を参照する。
- `AGENTS.md` チェックは `bash toolbox/skills/bootstrap-repository/scripts/check-agents-md.sh AGENTS.md` を実行する。

## 変更可能範囲

- このリポジトリ内のファイルは変更可能とする。
- 既存の作成済みスクリプトを使った変更は許可する。

## 変更注意範囲

- リポジトリ外のファイル変更は注意範囲とする。
- リポジトリ外へアクセスまたは変更する場合は、事前確認を必須とする。

## Done条件

- 依頼された変更が反映されていること。
- 関連する検証コマンドが成功していること。
- 実施した検証結果を報告していること。

## 報告フォーマット

- 報告には `実施内容`、`変更箇所`、`成功した検証`、`失敗した検証`、`判断が必要な点`、`次の推奨タスク` を含める。

## Git / 安全性ルール

- `main` への直接 `push` を禁止し、変更は PR 経由で反映する。
- 破壊的操作は明示依頼がある場合のみ許可する。
- `git reset --hard`、履歴改変、強制上書き、削除を伴う大規模変更などの破壊的操作は、明示依頼がある場合を除いて禁止する。
- 自分が加えていない差分を勝手に戻してはならない。
- 外部反映を伴う操作を行った場合は、実施内容を報告に明記する。

## 禁止事項

- リポジトリ外の変更を、事前確認なしで実施してはならない。
- 秘密情報、トークン、認証情報を追加・出力・コミットしてはならない。
- 検証未実施のまま完了扱いで報告してはならない。

## ログ運用

- 作業記録の正本はPR本文とする。
- `ai_log` は PR 化しない調査メモ、一時検証、一次対応に限って使用する。

## ファイル命名ルール

- 新規ファイル名の単語区切りはハイフンを使う。
- 新規ファイル名は `kebab-case` を使う。
- 新規ファイル名でアンダースコアは使わない。
- 既存ファイルは互換性のため維持してよい。
