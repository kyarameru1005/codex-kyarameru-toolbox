# Requirements

この文書は、リポジトリ整備時に満たすべき最低要求をまとめる。

## 運用要求

- `AGENTS.md` を最上位の運用ルールとして維持する。
- `README.md` から主要コマンドへ到達できるようにする。
- `docs/` に詳細文書を集約し、`AGENTS.md` へ長い手順を書き込まない。
- 既存スクリプトと実際に存在するコマンドだけを案内する。

## 構造要求

- `docs/` は仕様、研究メモ、運用文書の置き場とする。
- `ai_log/` は PR 化しない調査メモ、一時検証、一次対応の記録に限定する。
- `tests/` は自動検証コードと、その補助説明の置き場とする。
- 新規ファイル名は `kebab-case` を使う。

## 品質要求

- 変更後は関連する検証コマンドを実行する。
- 秘密情報や `.env` を追加しない。
- 既存ファイルを無断で上書きしない。
- 研究記録と運用手順を混在させない。
