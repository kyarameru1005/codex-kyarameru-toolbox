# Agents Rules

## Repository Rules
- リポジトリ外の破壊的操作は禁止する。
- リポジトリ外の変更が必要な場合は、事前に必要性と影響範囲を説明して確認を取る。
- 依頼範囲外のリファクタや大量整形は行わない。
- 実データや秘密情報を新たにコミットしない。
- 差分共有やコミット前に、ローカル固有情報や他リポジトリの情報が漏れていないか確認する。
- `toolbox/` 配下のデータが設定か生成物か不明な場合は、確認してから編集する。
- テスト未実行のまま完了扱いにしない。
- エラーを無視して成功扱いにしない。

## Editing Policy
- 変更は依頼達成に必要な最小限にとどめる。
- 既存の構成、命名、文体に合わせる。
- README や `docs/` と実装がずれた場合は、関連文書も更新する。
- バックアップ、キャッシュ、ログなどの生成物には無関係な変更を加えない。
