# GPT-5.5 Prompt Guide

## 1.目的
達成したい目標を定義する

## 2.前提
技術スタックや状況を伝える

## 3.制約
変えてはいけない条件

## 4.出力形式
どのように書いて欲しいか

## 5.完了条件
何ができたらOKか

## 具体例
~~~markdown
目的:
既存のFastAPIプロジェクトにヘルスチェックAPIを追加する。

前提:
- Python 3.12
- FastAPI
- uv使用
- 既存構成は変更しない

実装内容:
- GET /healthz を追加
- レスポンスは {"status": "ok"}
- テストを追加
- 既存テストが通ることを確認

制約:
- 不要な依存関係を追加しない
- 既存APIの挙動を変えない
- 変更ファイルを最小限にする

出力:
1. 変更方針
2. 変更ファイル一覧
3. 実装コード
4. テストコード
5. 実行コマンド
~~~
