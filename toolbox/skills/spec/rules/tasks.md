# Tasks Phase Rules

## 目的

タスク分解。全成果物（design.md + arch-check.md + test-spec.md）をもとに、実装可能な粒度にタスクを分割し、実装フェーズで迷わない順序を決定する。

## 入力

- design.md（実装対象の詳細設計）
- arch-check.md（最適化指摘・推奨アクション）
- test-spec.md（テスト仕様・テストファイル構成）

## 出力

- tasks.md

## タスク粒度基準

| サイズ | 目安 | 判定 |
|--------|------|------|
| XS | 〜30分 | ✓ 理想 |
| S | 〜1時間 | ✓ 推奨 |
| M | 〜2時間 | ✓ 許容 |
| L | 〜4時間 | △ 分割検討 |
| XL | 4時間〜 | ✗ 必ず分割 |

## Phase 構成（TDD 対応）

```mermaid
graph LR
    P1["Phase 1: テストコード作成 (RED)"]
    P2["Phase 2: ドメイン層 (GREEN)"]
    P3["Phase 3: サービス層 (GREEN)"]
    P4["Phase 4: ハンドラ層 (GREEN)"]
    P5["Phase 5: その他実装 (GREEN)"]
    P6["Phase 6: ドキュメント"]
    P7["Phase 7: 最終検証 (REFACTOR)"]
    P1 --> P2 --> P3 --> P4 --> P5 --> P6 --> P7
```

- **Phase 1**: test-spec.md に基づきテストコードを作成（全テスト FAIL を確認）
- **Phase 2-5**: テストを通す実装（各 Phase 完了時にテスト実行必須）
- **Phase 6**: ドキュメント更新
- **Phase 7**: 全テスト通過 + arch-check の推奨アクション反映確認 + テスト修正履歴の正当性確認

## タスク生成ルール

### test-spec.md からの導出
- Phase 1 のテストコード作成タスクは test-spec.md のテストファイル構成から直接導出
- 各テストファイルごとに 1 タスク（粒度が大きければ分割）

### design.md からの導出
- Phase 2-5 の実装タスクは design.md の実装戦略（§8）から導出
- ファイル構成に合わせてタスクを分割

### arch-check.md からの導出
- 「推奨アクション」の必須項目は対応するタスクとして含める
- パフォーマンス最適化項目は Phase 5 または Phase 7 に配置

## 成功条件

- 全タスク: L 以下のサイズ
- 各 Phase: 最低 1 タスク
- 依存関係: 明示的に定義
- テストタスク: Phase 1 に含まれている
- 各実装 Phase (2-5): テスト実行が明記されている
- arch-check の必須アクション: 対応タスクが存在

## 停止条件

| 条件 | 対応 |
|------|------|
| XL タスクが存在 | 分割を提案 |
| 依存関係が循環 | 設計見直しを提案 |
| design.md に情報不足 | design に戻る |
| test-spec.md に情報不足 | test-spec に戻る |

## 遷移条件 → 承認

- 全タスク L 以下
- 依存関係に循環なし
- 全 Phase にタスクあり
- arch-check の必須アクションが全てタスク化されている

## Good/Bad パターン

```markdown
❌ Bad:
  - [ ] ユーザー管理機能を実装（XL - 曖昧、巨大）

✅ Good:
  - [ ] TASK-001: テストコード作成（S）
    - test-spec.md に基づきテストファイルを作成
    - _依存: なし_
  - [ ] TASK-002: User エンティティ作成（XS）
    - _依存: TASK-001_
  - [ ] TASK-003: UserRepository インターフェース定義（XS）
    - _依存: TASK-002_
  - [ ] TASK-004: UserRepository PostgreSQL 実装（S）
    - _依存: TASK-003_
  - [ ] TASK-005: CreateUser ユースケース実装（M）
    - _依存: TASK-004_
```

## タスク命名規則

```
TASK-{issue番号}-{連番}: {動詞}{目的語}（{サイズ}）
```

例: `TASK-123-001: テストコード作成（S）`

## チェックリスト

- [ ] 全タスクに TASK-XXX-YYY 形式の ID 付与
- [ ] 全タスクにサイズ（XS〜L）を明記
- [ ] XL タスクがない
- [ ] 依存関係に循環がない
- [ ] Phase 1〜7 すべてにタスクあり
- [ ] Phase 1 にテストコード作成タスクあり
- [ ] Phase 2-5 の各タスクにテスト実行が明記されている
- [ ] arch-check の必須アクションが対応タスクに含まれている
