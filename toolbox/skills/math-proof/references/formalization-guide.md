# 自然言語→形式証明 変換ガイド

## 変換の基本原則

1. **主張の形式化**: 自然言語の定理を論理式で表現
2. **定義の展開**: 使用される数学的定義を明示
3. **推論の形式化**: 各ステップの論理規則を明記
4. **仮定の追跡**: どの仮定がどこで使われるかを明示

## 変換プロセス

### Step 1: 主張の翻訳

自然言語 → 述語論理式

| 自然言語 | 形式化 |
|----------|--------|
| 「すべての偶数は2で割り切れる」 | $\forall n \in \mathbb{Z}: \text{even}(n) \implies 2 \mid n$ |
| 「素数は無限に存在する」 | $\forall N \in \mathbb{N}: \exists p > N: \text{prime}(p)$ |
| 「連続関数は中間値を取る」 | $\forall f \in C[a,b]: \forall y \in [\min(f(a),f(b)), \max(f(a),f(b))]: \exists c \in [a,b]: f(c) = y$ |

### Step 2: 定義の展開

各述語を基本的な定義まで展開:

```
even(n) := ∃k ∈ ℤ: n = 2k
prime(p) := p > 1 ∧ ∀d ∈ ℤ: (d | p) → (d = 1 ∨ d = p)
continuous(f, a) := ∀ε > 0: ∃δ > 0: ∀x: |x - a| < δ → |f(x) - f(a)| < ε
```

### Step 3: 証明の形式化

各行を「番号. 主張 [根拠]」の形で記述:

```
1. P                        [仮定]
2. P → Q                    [定理 X]
3. Q                        [1, 2, →除去]
4. Q → R                    [仮定]
5. R                        [3, 4, →除去]
```

### Step 4: 依存関係の検証

- 各行は前の行と公理・定理のみに依存
- 仮定のスコープを明確に管理
- 放電（discharge）される仮定を追跡

## フォーマット

### Fitch スタイル

```
1  │ n は偶数                    [仮定]
2  │ ∃k: n = 2k                 [1, 偶数の定義]
3  │ n = 2k₀                    [2, ∃除去]
4  │ n² = 4k₀²                  [3, 代数]
5  │ n² = 2(2k₀²)               [4, 因数分解]
6  │ ∃m: n² = 2m                [5, ∃導入]
7  │ n² は偶数                   [6, 偶数の定義]
8  n が偶数 → n² が偶数          [1-7, →導入]
```

### シーケント計算スタイル

```
                    Γ, P ⊢ Q
    ────────────────────────── →導入
         Γ ⊢ P → Q
```

## 一般的な変換パターン

| 自然言語パターン | 形式化パターン |
|----------------|--------------|
| 「〜とする」「〜を仮定する」 | 仮定の導入 |
| 「〜の定義より」 | 定義の展開 |
| 「〜から〜が従う」 | →除去（モーダスポネンス） |
| 「したがって」「ゆえに」 | 推論ステップ |
| 「任意の〜について」 | ∀導入 |
| 「〜が存在する」 | ∃導入 |
| 「場合分けをする」 | ∨除去 |
| 「矛盾」 | ⊥の導出 |
