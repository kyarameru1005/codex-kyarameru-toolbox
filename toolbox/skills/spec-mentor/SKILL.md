# Spec Mentor - オンデマンド学習

## 使い方

```
/spec mentor "なぜ TDD で書くのか"
/spec mentor "SOLID 原則を実例で説明して"
/spec mentor "この PR のアーキテクチャ判断を解説して"
/spec mentor "p値って何？この実験結果をどう読む？"
```

## 解説フォーマット

以下の構造で回答する:

### 1. 一言で答え（30秒で理解）

> {概念の核心を1-2文で}

### 2. なぜ重要か（ビジネスインパクト）

{この概念を無視するとどうなるか、実際の事例}

### 3. 具体例（このプロジェクトのコードで）

{可能であれば、現在のプロジェクトのコードから例を引用}
{プロジェクトにない場合は、一般的な例}

### 4. よくある間違い

{ジュニアエンジニアがやりがちなアンチパターン}

### 5. もっと学ぶには

{書籍、記事、動画の推薦 — 1-2個}

## 対応トピック

### アーキテクチャ・設計
- SOLID 原則（各原則の実践的な意味）
- レイヤードアーキテクチャ（各層の責務）
- DDD 基本概念（エンティティ、値オブジェクト、集約）
- C4 Model（いつどのレベルを使うか）
- マイクロサービス vs モノリス

### テスト
- TDD（RED-GREEN-REFACTOR の実践）
- テストピラミッド（単体/統合/E2E の使い分け）
- テストダブル（モック/スタブ/スパイの違い）
- プロパティベーステスト

### データ・統計（/spec research 向け）
- 統計的有意性（p値、信頼区間）
- A/B テスト設計（サンプルサイズ計算）
- 効果量（Cohen's d）
- 相関 vs 因果

### プロダクト開発
- Build-Measure-Learn ループ
- ICE フレームワーク
- Feature Flag パターン
- North Star メトリクス

### セキュリティ
- OWASP Top 10
- 認証 vs 認可
- SQL Injection / XSS 防止
- 最小権限の原則

## 知識ソース

以下の既存スキルの知識を活用:
- `senior-architect` — 設計原則、references/ ディレクトリの全リファレンス
- `paper-analysis` — 論文分析の方法論
- `spec-verify-matrix` — テストヒューリスティクス（SFDIPOT、データ型攻撃）
- `WebSearch` — 最新の情報、ベストプラクティス

## コンテキスト活用

可能であれば、現在のプロジェクトのコードを参照して具体例を示す:
1. Glob/Grep で関連コードを検索
2. 実際のコードを引用して「ここでこの原則が使われています」と示す
3. 改善点があれば「こうするともっと良くなります」と提案
