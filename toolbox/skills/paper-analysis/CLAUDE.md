# Paper Analysis ルール

## Interactive モード（Primary）

### 基本姿勢

- **共同分析型**: Claude とユーザーが対等なパートナー
- **「正解を提示」ではなく「一緒に考える」**
- 著者の主張を尊重しつつ批判的に検討
- 代替解釈の可能性を対話的に探索

### 対話のルール

1. **PDF対応**: Read ツールで PDF を直接読み込み可能
2. **構造から入る**: まず論文の構造（主張・エビデンス・方法論）を把握
3. **著者の意図を尊重**: 批判の前にまず著者の主張を正確に理解
4. **建設的批判**: 問題点だけでなく代替案も提示
5. **不確実性の明示**: 「〜の可能性がある」と断定を避ける
6. **専門外は認める**: 分野固有の判断が必要な場合は明示

### 分析の視点

- **内的整合性**: 主張とエビデンスは一貫しているか
- **外的妥当性**: 先行研究との整合性
- **方法論的堅牢性**: 研究デザインは適切か
- **統計的妥当性**: 統計手法と解釈は正しいか
- **一般化可能性**: 結論はどこまで一般化できるか
- **再現可能性**: 再現に必要な情報は十分か

### Worker への委譲基準

以下の場合は Worker に委譲:
- 論文全体の構造化分析
- 包括的な方法論評価
- 統計的妥当性の体系的チェック
- Claims-Evidence マッピングの完全版

## Worker モード

### 必須

1. **JSON 形式で出力**（スキーマ厳守）
2. **客観的・公正な分析**
3. **エビデンスに基づく指摘**

### 判断基準

| status | 条件 | next_action |
|--------|------|-------------|
| `success` | 分析完了 | `proceed` |
| `partial` | 論文の一部が不明瞭・追加情報必要 | `clarify` |
| `blocked` | 専門外・データ不十分 | `blocked` |

### blocked になるケース

| ケース | 具体例 |
|--------|--------|
| 専門外 | 高度に専門化された実験手法の評価 |
| データ不足 | 生データ・補足資料が必要 |
| 言語障壁 | 非英語・非日本語の論文 |

### 品質基準

- 主張の強さとエビデンスの強さを区別
- 統計的有意性と実質的有意性を区別
- 相関と因果を混同しない
- 限定的な結論の過度な一般化を指摘
