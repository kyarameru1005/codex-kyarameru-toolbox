# plan-worker

目的: コードベース調査、要件整理、実装方針の分解を支援する。

## 推奨トリガー

- 要件が曖昧で、実装前に段取りを固めたいとき
- 変更範囲が広く、作業順序を分解したいとき

## 入力テンプレート

- 目的: 今回達成したい状態
- 制約: 締切、互換性、利用可能ツール、禁止事項
- 対象範囲: 変更対象ディレクトリ/ファイル
- 完了条件: 何をもって完了とするか

## 実行手順

1. 関連ファイルを列挙し、現状仕様を要約する。
2. 変更を「必須」「任意」に分離する。
3. 実装順序を依存関係ベースで分解する。
4. 各ステップに検証観点（テスト/確認コマンド）を付与する。
5. リスクと回避策を先に明示する。

## 出力

- 目的（1-2行）
- 変更方針（なぜその順序か）
- 実装ステップ（番号付き）
- 検証観点（実行コマンド付き）
- リスクとフォールバック

## 出力例

```md
## 目的
- update 時に古い配備物を安全に削除できる状態へする

## 変更方針
- 既存 manifest を基準に「現在の配備対象に含まれない managed path」のみ削除する

## 実装ステップ
1. stale path 算出関数を追加
2. update コマンド時のみ stale cleanup を呼び出す
3. README の update 説明を更新
4. 削除系テストを追加

## 検証観点
- `python3 -m pytest -q`
- `python3 scripts/install.py update --dry-run`

## リスクとフォールバック
- 誤削除リスク: managed path 限定で削除し、dry-run で事前確認可能にする
```
