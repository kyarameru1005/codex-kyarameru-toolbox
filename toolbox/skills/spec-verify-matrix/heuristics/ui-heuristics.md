# UI テスト ヒューリスティクス

## ナビゲーション検証

- [ ] ブラウザの戻る/進むボタンで状態が壊れないか
- [ ] ページリロードで状態が保持されるか
- [ ] ブックマーク URL で直接アクセスできるか
- [ ] ログアウト状態でのブックマーク URL アクセス → ログインへリダイレクト
- [ ] URL パラメータの手動変更で不正アクセスできないか
- [ ] 複数タブで同時操作しても整合性が保たれるか
- [ ] ディープリンクが正しく動作するか

## フォーム検証

- [ ] 空送信時の適切なバリデーションメッセージ
- [ ] バリデーションエラー時のフォーカス移動
- [ ] 二重送信防止（ボタン無効化 or デバウンス）
- [ ] 入力途中でのページ離脱警告
- [ ] コピペ入力でもバリデーションが動作するか
- [ ] オートコンプリート/オートフィルとの互換性
- [ ] 入力マスク（電話番号、郵便番号等）の動作

## アクセシビリティ (WCAG 2.1 AA)

### キーボード操作
- [ ] Tab キーで全インタラクティブ要素に到達可能
- [ ] フォーカス順序が論理的（左→右、上→下）
- [ ] フォーカスインジケーターが視認可能
- [ ] キーボードトラップがない（モーダル内はOK、ESCで脱出可能）
- [ ] Enter/Space でボタン・リンクが動作する
- [ ] カスタムコンポーネントのキーボード操作

### スクリーンリーダー
- [ ] 全画像に適切な alt 属性（装飾画像は alt=""）
- [ ] フォーム要素に紐づいた label
- [ ] ARIA ランドマーク（main, nav, footer 等）
- [ ] 動的コンテンツ更新時の aria-live 通知
- [ ] ページの lang 属性が正しい

### 視覚
- [ ] テキストと背景のコントラスト比 4.5:1 以上
- [ ] 色だけに依存しない情報伝達（アイコン + テキスト）
- [ ] 200% ズームでコンテンツが切れない
- [ ] アニメーションの停止/減速が可能（prefers-reduced-motion）

## レスポンシブデザイン

| ブレークポイント | 幅 | 確認項目 |
|----------------|-----|---------|
| モバイル | 320px - 480px | タッチターゲット44px以上、横スクロールなし |
| タブレット | 768px - 1024px | レイアウト切替、ナビゲーション変化 |
| デスクトップ | 1280px+ | 全機能表示、ホバー操作 |
| 縦横回転 | - | orientation change でレイアウト崩れなし |

## クロスブラウザ

| ブラウザ | 確認項目 |
|---------|---------|
| Chrome (最新) | 基準動作 |
| Firefox (最新) | CSS Grid/Flexbox 差異、フォントレンダリング |
| Safari (最新) | date input、CSS backdrop-filter、iOS 固有の挙動 |
| Edge (最新) | Chrome と概ね同等だが独自の差異 |

## パフォーマンス

- [ ] LCP (Largest Contentful Paint) < 2.5秒
- [ ] FID (First Input Delay) < 100ms
- [ ] CLS (Cumulative Layout Shift) < 0.1
- [ ] 画像の遅延読み込み（viewport外）
- [ ] バンドルサイズの確認
- [ ] コンソールエラーがないこと

## エラー状態の UX

- [ ] ネットワークエラー時の適切な表示
- [ ] ローディング中の表示（スケルトン or スピナー）
- [ ] 空状態（データなし）の表示
- [ ] 404 ページ
- [ ] 500 エラー時のフォールバック表示
- [ ] 入力バリデーションエラーの表示位置と文言
