# SFDIPOT - 探索的テストの7軸

各機能・コンポーネントに対して以下の7軸を体系的に適用する。

## S - Structure（構造）

構造的弱点を探す:
- コンポーネント間の依存関係は正しいか
- データフローに漏れはないか
- モジュール境界は明確か
- 循環依存はないか
- エラー伝搬パスは適切か

## F - Function（機能）

機能の正しさを検証する:
- **正常系**: 仕様通りの入力 → 期待通りの出力
- **異常系**: 不正入力 → 適切なエラー処理
- **境界値**: 最小値、最大値、丁度の値、±1
- **CRUD**: Create/Read/Update/Delete の全操作
- **Follow the Data**: 入力→処理→保存→取得→表示→エクスポートの全経路

## D - Data（データ）

データの扱いを検証する:
- 空データ（null, undefined, 空文字, 空配列）
- 巨大データ（上限値、上限超過）
- 不正データ（型不一致、フォーマット違反）
- 特殊文字（SQL injection, XSS, 絵文字, マルチバイト）
- ユニーク制約違反（重複データ）
- 参照整合性（存在しないID参照）

## I - Interface（インターフェース）

外部との接点を検証する:
- API コントラクト（リクエスト/レスポンス形式）
- 認証/認可の境界
- 外部サービス連携（タイムアウト、障害時）
- ファイルI/O（読み書き権限、同時アクセス）
- データベース接続（接続断、プール枯渇）

## P - Platform（プラットフォーム）

動作環境を検証する:
- ブラウザ差異（Chrome, Firefox, Safari, Edge）
- OS 差異（Windows, macOS, Linux）
- 画面サイズ（モバイル, タブレット, デスクトップ）
- ネットワーク（高速, 低速, オフライン）
- リソース（メモリ不足, ディスク不足）

## O - Operations（運用）

実運用条件を検証する:
- 同時接続（マルチユーザー同時操作）
- 長時間稼働（メモリリーク、コネクション漏れ）
- リソース枯渇（CPU, メモリ, ディスク, コネクション）
- バックアップ/リストア
- ログ出力（適切な情報、機密情報漏洩なし）
- 監視（ヘルスチェック、メトリクス）

## T - Time（時間）

時間に関わるものを検証する:
- タイムアウト（リクエスト、セッション、トークン）
- タイムゾーン（UTC, JST, 夏時間）
- 日付境界（月末、年末、閏年）
- 同時実行（レースコンディション、デッドロック）
- 順序依存（イベント順序、メッセージ順序）
- スケジュール（定期実行、遅延実行）
