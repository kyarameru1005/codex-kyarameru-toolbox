# API テスト ヒューリスティクス

## BINMEN（APIテストの6軸）

各APIエンドポイントに対して適用する。

### B - Boundary（境界値）
- リクエストパラメータの最小/最大値
- ページネーションの境界（page=0, page=MAX）
- 配列パラメータの要素数（0, 1, 1000+）
- 文字列長の上限/下限

### I - Invalid Entries（不正入力）
- 型不一致（数値に文字列、オブジェクトにスカラー）
- 必須パラメータの欠落
- 未知のパラメータの追加
- フォーマット違反（不正なメールアドレス、UUID等）

### N - NULL
- null値の送信（各フィールド）
- undefined / 欠落フィールド
- 空文字 vs null の区別
- ネストされたオブジェクト内のnull

### M - Method（HTTPメソッド）
- 許可されていないメソッド → 405
- OPTIONS プリフライト対応
- HEAD リクエスト対応
- PATCH vs PUT の挙動差

### E - Empty（空）
- 空のリクエストボディ
- 空の配列 `[]`
- 空のオブジェクト `{}`
- Content-Length: 0

### N - Negative（否定・逆）
- 負の数値ID
- 逆順ソート
- 存在しないリソースID → 404
- 削除済みリソースへのアクセス

## VADER（API品質の5軸）

### V - Verbs（HTTPメソッド適切性）
- GET は副作用なし（冪等性）
- POST は新規作成 → 201
- PUT は全体更新（冪等性）
- PATCH は部分更新
- DELETE → 204 or 200

### A - Auth（認証・認可）
- 認証なしアクセス → 401
- 権限不足アクセス → 403
- 期限切れトークン → 401
- 他ユーザーのリソースアクセス → 403 or 404
- CORS（許可/非許可オリジン）

### D - Data（データ整合性）
- レスポンス形式（JSON Schema準拠）
- 必須フィールドの存在
- 日付フォーマットの一貫性
- ページネーションメタデータ
- エンベロープの一貫性

### E - Errors（エラーハンドリング）
- エラーレスポンス形式の統一
- 適切なHTTPステータスコード
- エラーメッセージに内部情報漏洩なし
- バリデーションエラーのフィールド特定
- 複数エラーの一括返却

### R - Responsiveness（応答性）
- レスポンスタイム（p50, p95, p99）
- 大量データ時のパフォーマンス
- N+1クエリの検出
- レートリミット動作（429）

## セキュリティ検証

### 認証セキュリティ
- [ ] ブルートフォース防止（アカウントロック / レート制限）
- [ ] パスワードポリシー（最小長、複雑性）
- [ ] セッション管理（タイムアウト、無効化）
- [ ] トークン管理（有効期限、ローテーション）

### 入力セキュリティ
- [ ] SQL injection 防止
- [ ] NoSQL injection 防止
- [ ] Command injection 防止
- [ ] XSS 防止（Stored, Reflected, DOM-based）
- [ ] SSRF 防止
- [ ] パストラバーサル防止

### ヘッダーセキュリティ
- [ ] X-Content-Type-Options: nosniff
- [ ] X-Frame-Options: deny
- [ ] Content-Security-Policy 設定
- [ ] Strict-Transport-Security (HSTS)
- [ ] サーバーバージョン非公開

### データセキュリティ
- [ ] 機密情報がレスポンスに含まれない（パスワードハッシュ等）
- [ ] ログに機密情報が出力されない
- [ ] エラーメッセージにスタックトレースなし（本番）
- [ ] 適切な暗号化（TLS 1.2+）
