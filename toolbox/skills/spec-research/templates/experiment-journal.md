# Experiment Journal

プロダクトの全実験記録。AI が自動更新し、人間が後から学びを追跡するための累積ドキュメント。

## 読み方

- 各実験は時系列で記録される
- `学び` セクションが最も重要 — 後続の仮説生成で参照される
- `[CONFIRMED]` = 仮説が正しかった、`[REJECTED]` = 仮説が間違っていた、`[INCONCLUSIVE]` = データ不足

---

## 記録フォーマット

各実験完了時に以下の形式で自動追記される:

```
## EXP-{date}-{seq}: {name}
- 仮説: {hypothesis}
- 根拠: {evidence}
- 変更内容: {changes}
- 影響ファイル: {files}
- 結果: {result}
- Confidence: {score}

### 学び
- **技術概念**: {この実験で使った/学んだ技術概念}
- **設計判断**: {なぜこの設計を選んだか}
- **テスト手法**: {この実験で使ったテスト手法}
- **データ分析**: {分析手法とその結果の読み方}

### あなたが実装した部分（spec-apprentice モードの場合）
- `{関数名}`: {何を学んだか}
- 結果: {テスト通過/修正が必要だった点}

### 次に活かすこと
- {次の実験や実装で活用できるポイント}
```
