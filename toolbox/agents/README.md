# agents

このディレクトリは、サブエージェント定義を置く場所です。
各 `*.toml` は 1 つのエージェント設定を表し、役割、使用モデル、推論強度、実行権限、行動指針を持ちます。

## 基本方針

- 1つのエージェントに複数の責務を詰め込まない。
- 調査、設計、実装、レビュー、検証の責務を分ける。
- まず必要最小限の役割だけを作り、必要になったら増やす。

## 典型的な役割

- 判断役: 何を自分で処理し、何を委譲するかを決める。
- 調査役: 関連ファイル、依存関係、影響範囲を整理する。
- 設計役: 方針、責務分離、拡張性を整理する。
- 実装役: 設計に沿って変更を加える。
- 評価役: レビュー、テスト、リスク確認を行う。

## オーケストレーション

- メインの Codex は全体判断と依頼の分配を担当する。
- 関連ファイルや影響範囲が不明なら、先に調査役を使う。
- 設計判断が必要なら、実装前に設計役を使う。
- 実装後はレビューや検証を別役割で確認する。
- 並列化は独立した作業だけに限定し、同じファイルを複数エージェントで同時に触らせない。

## 注意点

- 書き込み権限を持つエージェントは最小限にする。
- 読み取り専用エージェントは調査や評価に向いている。
- 役割名は自由だが、責務が名前から分かる状態を保つ。
